# 🔷 1. What are Exceptions?

* Events that disrupt normal program flow
* Handled using **Exception Handling mechanism**


# 🔷 2. Classification of Exceptions

```text
Exception
 ├── Checked Exceptions
 └── Unchecked Exceptions (RuntimeException)
```


# 🔷 3. Checked Exceptions (Compile-Time) ⭐

## 🔹 Definition

* Exceptions checked by **compiler at compile time**
* **Must be handled** using `try-catch` OR `throws`


## 🔹 Examples

* `IOException`
* `SQLException`
* `FileNotFoundException`


## 🔹 Example Code

```java
import java.io.*;

class Test {
    public static void main(String[] args) {
        try {
            FileReader fr = new FileReader("file.txt");
        } catch (FileNotFoundException e) {
            System.out.println("File not found");
        }
    }
}
```


## 🔹 Key Points

* Compiler forces handling
* Safer, prevents unexpected crashes
* Used for **external resources** (files, DB, network)


## 🔹 If Not Handled

👉 Compile-time error

```java
FileReader fr = new FileReader("file.txt"); // ERROR
```


# 🔷 4. Unchecked Exceptions (Runtime) ⭐

## 🔹 Definition

* Occur at **runtime**
* Not checked by compiler
* Handling is **optional**


## 🔹 Examples

* `NullPointerException`
* `ArithmeticException`
* `ArrayIndexOutOfBoundsException`


## 🔹 Example Code

```java
class Test {
    public static void main(String[] args) {
        int a = 10 / 0; // ArithmeticException
    }
}
```


## 🔹 Key Points

* Caused by **programming mistakes**
* Not mandatory to handle
* Occur during execution


# 🔷 5. Checked vs Unchecked (Core Differences 🔥)

| Feature    | Checked Exception   | Unchecked Exception        |
| ---------- | ------------------- | -------------------------- |
| Checked at | Compile time        | Runtime                    |
| Handling   | Mandatory           | Optional                   |
| Package    | java.lang.Exception | java.lang.RuntimeException |
| Cause      | External factors    | Programming errors         |
| Examples   | IOException         | NullPointerException       |


# 🔷 6. Hierarchy Difference

```text
Throwable
 ├── Exception
 │     ├── Checked Exceptions
 │     └── RuntimeException (Unchecked)
```

👉 All **unchecked exceptions extend RuntimeException**

# 🔷 7. Interview Traps ⚠️

## ✔️ Trap 1: Compiler Check

```java
FileReader fr = new FileReader("file.txt");
```

👉 ❌ Compile-time error → Checked exception

## ✔️ Trap 2: Runtime Crash

```java
String s = null;
System.out.println(s.length());
```

👉 ❌ NullPointerException (runtime)

---

## ✔️ Trap 3: Handling Not Mandatory

```java
int a = 10 / 0;
```

👉 Compiles fine, crashes at runtime

---

# 🔷 8. When to Use What? (INTERVIEW ANSWER ⭐)

## ✔️ Use Checked Exception:

* File handling
* Database operations
* Network calls

👉 Because failure is **recoverable**

---

## ✔️ Use Unchecked Exception:

* Logical errors
* Validation errors
* Bugs in code

👉 Because it’s a **programmer mistake**

---

# 🔷 9. Custom Exception Type Decision

* Extend `Exception` → Checked
* Extend `RuntimeException` → Unchecked

```java
class MyException extends Exception {}          // Checked
class MyException2 extends RuntimeException {} // Unchecked
```

---

# 🔷 10. Best Practices ✅

* Don’t overuse checked exceptions
* Use unchecked for logic errors
* Always handle checked exceptions properly
* Avoid catching generic `Exception`

---

# 🔷 11. One-Line Difference (Perfect for Interview 🎯)

👉 *"Checked exceptions are verified at compile time and must be handled, whereas unchecked exceptions occur at runtime and are usually caused by programming errors."*
