# 🔥 Types of Methods in Java

## 1️⃣ Instance Method (Non-static)

### ✅ Theory

* Belongs to an **object**
* Requires object to call
* Can access **both instance and static variables**

---

### 📌 Code

```java
class Test {
    void show() {
        System.out.println("Instance method");
    }

    public static void main(String[] args) {
        Test t = new Test();
        t.show();
    }
}
```

### 💡 Explanation

* Object `t` is created
* Method is called using object → `t.show()`

---

## 2️⃣ Static Method (Class Method)

### ✅ Theory

* Belongs to **class**
* No object required
* Can access **only static members directly**

---

### 📌 Code

```java
class Test {
    static void display() {
        System.out.println("Static method");
    }

    public static void main(String[] args) {
        display();
    }
}
```

### 💡 Explanation

* Method called directly without object
* Runs when class is loaded

---

## 3️⃣ Abstract Method

### ✅ Theory

* Declared without body
* Must be implemented by subclass
* Used in **abstraction**

---

### 📌 Code

```java
abstract class A {
    abstract void show();
}

class B extends A {
    void show() {
        System.out.println("Implemented");
    }
}
```

### 💡 Explanation

* `show()` has no body in class A
* Class B provides implementation

---

## 4️⃣ Final Method

### ✅ Theory

* Cannot be overridden
* Used to restrict modification

---

### 📌 Code

```java
class A {
    final void show() {
        System.out.println("Final method");
    }
}
```

### 💡 Explanation

* Any subclass cannot override `show()`

---

## 5️⃣ Synchronized Method

### ✅ Theory

* Used in **multithreading**
* Allows only one thread at a time

---

### 📌 Code

```java
class Test {
    synchronized void show() {
        System.out.println("Thread-safe method");
    }
}
```

### 💡 Explanation

* Prevents multiple threads from accessing method simultaneously

---

## 6️⃣ Static vs Instance Method (Quick Difference)

| Feature      | Static | Instance |
| ------------ | ------ | -------- |
| Belongs to   | Class  | Object   |
| Needs object | ❌ No   | ✅ Yes    |

---

# 🎯 Quick Summary

* **Instance method** → object-based
* **Static method** → class-based
* **Abstract method** → no body
* **Final method** → cannot override
* **Synchronized method** → thread safety
