# 🔷 1. What is File Handling?

* Process of **creating, reading, writing, and deleting files**
* Java provides APIs in:

  * `java.io` (old/traditional)
  * `java.nio` (modern, faster)


# 🔷 2. What are Streams?

* A **stream** is a flow of data between source and destination

👉 Two types:

```text
Input Stream  → Read data
Output Stream → Write data
```


# 🔷 3. Byte Streams vs Character Streams ⭐

| Feature   | Byte Streams                 | Character Streams |
| --------- | ---------------------------- | ----------------- |
| Data type | 8-bit bytes                  | 16-bit characters |
| Classes   | InputStream / OutputStream   | Reader / Writer   |
| Use       | Binary files (images, video) | Text files        |


# 🔷 4. Byte Streams (IMPORTANT)

## 🔹 InputStream (Read)

```java
FileInputStream fis = new FileInputStream("file.txt");
int data = fis.read();
```

## 🔹 OutputStream (Write)

```java
FileOutputStream fos = new FileOutputStream("file.txt");
fos.write(65);
```


## 🔹 Common Classes

* `FileInputStream`
* `FileOutputStream`
* `BufferedInputStream`
* `BufferedOutputStream`


# 🔷 5. Character Streams

## 🔹 Reader (Read)

```java
FileReader fr = new FileReader("file.txt");
int ch = fr.read();
```

## 🔹 Writer (Write)

```java
FileWriter fw = new FileWriter("file.txt");
fw.write("Hello");
```


## 🔹 Common Classes

* `FileReader`
* `FileWriter`
* `BufferedReader`
* `BufferedWriter`
* `PrintWriter`


# 🔷 6. Buffered Streams (Performance Boost 🚀)

```java
BufferedReader br = new BufferedReader(new FileReader("file.txt"));
String line = br.readLine();
```

👉 Benefits:

* Faster (reduces I/O operations)
* Reads data in chunks

# 🔷 8. Reading & Writing Example

## ✔️ Read File

```java
BufferedReader br = new BufferedReader(new FileReader("test.txt"));
String line;
while ((line = br.readLine()) != null) {
    System.out.println(line);
}
```


## ✔️ Write File

```java
BufferedWriter bw = new BufferedWriter(new FileWriter("test.txt"));
bw.write("Hello Java");
bw.close();
```


# 🔷 9. try-with-resources (VERY IMPORTANT ⭐)

```java
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    System.out.println(br.readLine());
}
```

👉 Automatically closes resources
👉 Prevents memory leaks


# 🔷 10. Serialization (INTERVIEW FAVORITE 🔥)

## 🔹 What is Serialization?

* Convert object → byte stream

```java
ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("obj.txt"));
oos.writeObject(obj);
```


## 🔹 Deserialization

```java
ObjectInputStream ois = new ObjectInputStream(new FileInputStream("obj.txt"));
Object obj = ois.readObject();
```


## 🔹 Important Points

* Class must implement `Serializable`
* `transient` → skip variable
* `serialVersionUID` → version control


# 🔷 11. Java NIO (New I/O) ⭐

## 🔹 Features

* Faster than `java.io`
* Non-blocking I/O
* Uses **Channels & Buffers**


## 🔹 Key Components

* `Path`
* `Files`
* `Channel`
* `Buffer`


## 🔹 Example

```java
Path path = Paths.get("file.txt");
Files.readAllLines(path);
```


# 🔷 12. Difference: IO vs NIO

| Feature     | IO           | NIO          |
| ----------- | ------------ | ------------ |
| Type        | Stream-based | Buffer-based |
| Blocking    | Blocking     | Non-blocking |
| Performance | Slower       | Faster       |


# 🔷 13. File Modes

```java
FileWriter fw = new FileWriter("file.txt", true); // append mode
```


# 🔷 14. Important Interview Concepts 🔥

## ✔️ 1. read() method

* Returns **int** (ASCII value)
* Returns **-1** at EOF


## ✔️ 2. Difference: FileReader vs BufferedReader

| FileReader         | BufferedReader     |
| ------------------ | ------------------ |
| Reads char by char | Reads line by line |
| Slower             | Faster             |


## ✔️ 3. Flush vs Close

* `flush()` → clear buffer
* `close()` → release resource


## ✔️ 4. Mark & Reset

* Used in streams to re-read data


# 🔷 15. Common Exceptions

* `IOException`
* `FileNotFoundException`
* `EOFException`


# 🔷 16. Best Practices ✅

* Always close streams (or use try-with-resources)
* Use **Buffered streams** for performance
* Prefer **NIO** for modern apps
* Handle exceptions properly


# 🔷 17. One-Line Interview Summary 🎯

👉 *"Java I/O provides stream-based and buffer-based mechanisms to efficiently perform file operations, with NIO offering high-performance, non-blocking capabilities."*
