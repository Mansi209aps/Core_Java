# 🔥 Polymorphism in Java


## 📌 1. Definition

**Polymorphism** means **“many forms”** — the same method name behaves differently based on context.

👉 Achieves:

* Flexibility
* Reusability
* Clean design

---

# 📌 2. Types of Polymorphism

## 🔹 1. Compile-Time Polymorphism (Static)

👉 Achieved using **Method Overloading**

---

### 📌 Example

```java id="r0w9bz"
class Test {

    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        Test t = new Test();
        System.out.println(t.add(2, 3));     // int
        System.out.println(t.add(2.5, 3.5)); // double
    }
}
```

---

## 💡 Explanation

* Same method name → different parameter types
* Method decided at **compile time**

---

---

## 🔹 2. Runtime Polymorphism (Dynamic)

👉 Achieved using **Method Overriding**

---

### 📌 Example

```java id="d5mhlr"
class Animal {
    void sound() {
        System.out.println("Animal sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }

    public static void main(String[] args) {
        Animal obj = new Dog();
        obj.sound();
    }
}
```

---

## 💡 Explanation

* Parent reference → child object
* Method decided at **runtime (JVM)**

---

# 📌 3. Key Concept: Upcasting

```java id="9txwwb"
Animal obj = new Dog();
```

👉 Parent reference pointing to child object

✔ Enables runtime polymorphism

---

# 📌 4. Method Binding

---

| Type          | When         | Example     |
| ------------- | ------------ | ----------- |
| Early Binding | Compile-time | Overloading |
| Late Binding  | Runtime      | Overriding  |

---

---

# 📌 5. Rules of Runtime Polymorphism

---

* Must use **inheritance**
* Method must be **overridden**
* Reference type = parent
* Object type = child

---

---

# 📌 6. Important Limitations

---

* Only **overridden methods** are polymorphic
* **Static, final, private methods** → NOT polymorphic
* Variables are NOT polymorphic

---

# 📌 10. Overloading vs Overriding

| Feature    | Overloading | Overriding   |
| ---------- | ----------- | ------------ |
| Time       | Compile     | Runtime      |
| Parameters | Different   | Same         |
| Class      | Same        | Parent-child |

---

# 🎯 Interview Summary

> Polymorphism in Java allows methods to take multiple forms. It is achieved through method overloading (compile-time polymorphism) and method overriding (runtime polymorphism). It enables flexibility and dynamic behavior in applications by allowing a single interface to represent different implementations.
