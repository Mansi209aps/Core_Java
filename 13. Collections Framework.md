# 🔷 1. What is Collections Framework?

* A **set of classes & interfaces** to store and manipulate groups of objects
* Located in `java.util` package

👉 Provides:

* Ready-made **data structures**
* Algorithms (sorting, searching)

---

# 🔷 2. Core Interfaces (VERY IMPORTANT 🔥)

```text
Iterable
   ↓
Collection
 ├── List
 ├── Set
 └── Queue
      └── Deque

Map (separate hierarchy)
```

---

# 🔷 3. Collection vs Collections (COMMON QUESTION ❗)

| Term          | Meaning                                    |
| ------------- | ------------------------------------------ |
| `Collection`  | Interface                                  |
| `Collections` | Utility class (methods like sort, reverse) |

---

# 🔷 4. List Interface (Ordered, Allows Duplicates)

## 🔹 Implementations:

### ✔️ ArrayList

* Dynamic array
* Fast random access
* Slow insertion/deletion (middle)

```java
List<Integer> list = new ArrayList<>();
```

---

### ✔️ LinkedList

* Doubly linked list
* Fast insertion/deletion
* Slow access

---

### ✔️ Vector (Legacy)

* Thread-safe
* Slower

---

### ✔️ Stack (Legacy)

* LIFO

```java
Stack<Integer> stack = new Stack<>();
```

---

## 🔹 Key Points

* Maintains **insertion order**
* Allows **duplicates**

---

# 🔷 5. Set Interface (No Duplicates)

## 🔹 Implementations:

### ✔️ HashSet

* Unordered
* Fast (O(1))

---

### ✔️ LinkedHashSet

* Maintains insertion order

---

### ✔️ TreeSet

* Sorted (ascending)
* Uses **Red-Black Tree**

---

## 🔹 Key Points

* No duplicates
* Uses `equals()` and `hashCode()`

---

# 🔷 6. Queue Interface (FIFO)

## 🔹 Implementations:

### ✔️ PriorityQueue

* Elements sorted by priority

---

### ✔️ LinkedList (also implements Queue)

---

## 🔹 Deque (Double-ended queue)

* Add/remove from both ends

```java
Deque<Integer> dq = new ArrayDeque<>();
```

---

# 🔷 7. Map Interface (Key-Value Pair) ⭐

## 🔹 Important:

* Not part of `Collection` interface

---

## 🔹 Implementations:

### ✔️ HashMap

* Unordered
* Allows 1 null key, multiple null values
* Fast (O(1))

---

### ✔️ LinkedHashMap

* Maintains insertion order

---

### ✔️ TreeMap

* Sorted by keys

---

### ✔️ Hashtable (Legacy)

* Thread-safe
* No null key/value

---

# 🔷 8. Hashing Mechanism (VERY IMPORTANT 🔥)

* Used in **HashMap, HashSet**
* Based on:

  * `hashCode()`
  * `equals()`

👉 Flow:

1. Compute hashCode
2. Find bucket
3. Handle collision (Linked List / Tree)

---

# 🔷 9. Comparable vs Comparator ⭐

## ✔️ Comparable (Natural sorting)

```java
class A implements Comparable<A> {
    public int compareTo(A o) {
        return this.id - o.id;
    }
}
```

---

## ✔️ Comparator (Custom sorting)

```java
Collections.sort(list, (a, b) -> a.age - b.age);
```

---

## 🔹 Difference

| Feature | Comparable  | Comparator |
| ------- | ----------- | ---------- |
| Package | java.lang   | java.util  |
| Method  | compareTo() | compare()  |
| Sorting | Natural     | Custom     |

# 🔷 12. Time Complexity (INTERVIEW MUST ⭐)

| Structure  | Access   | Insert   | Delete   |
| ---------- | -------- | -------- | -------- |
| ArrayList  | O(1)     | O(n)     | O(n)     |
| LinkedList | O(n)     | O(1)     | O(1)     |
| HashSet    | O(1)     | O(1)     | O(1)     |
| TreeSet    | O(log n) | O(log n) | O(log n) |
| HashMap    | O(1)     | O(1)     | O(1)     |

# 🔷 13. Synchronization

* **Not Thread-safe by default**
* Use:

  * `Collections.synchronizedList()`
  * `ConcurrentHashMap`

---

# 🔷 14. Fail-Fast vs Fail-Safe Iterators

| Type      | Behavior                                 |
| --------- | ---------------------------------------- |
| Fail-Fast | Throws `ConcurrentModificationException` |
| Fail-Safe | Works on clone (no exception)            |

# 🔷 15. Key Interview Differences 🔥

### ✔️ ArrayList vs LinkedList

* ArrayList → fast access
* LinkedList → fast insert/delete

---

### ✔️ HashMap vs Hashtable

* HashMap → not thread-safe
* Hashtable → thread-safe

---

### ✔️ HashSet vs TreeSet

* HashSet → unordered
* TreeSet → sorted

---

# 🔷 16. When to Use What? (INTERVIEW ANSWER ⭐)

* Use **ArrayList** → frequent access
* Use **LinkedList** → frequent insert/delete
* Use **HashSet** → uniqueness
* Use **TreeSet** → sorted data
* Use **HashMap** → fast key-value access

---

# 🔷 17. Best Practices ✅

* Override `equals()` & `hashCode()`
* Prefer **interfaces over implementation**
* Use generics (`List<String>`)
* Avoid legacy classes (Vector, Hashtable)

---
