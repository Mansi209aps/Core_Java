# 🔥 Inheritance in Java

## 📌 1. Definition

**Inheritance** is a mechanism where one class (**child/subclass**) acquires properties and behavior of another class (**parent/superclass**).

👉 Achieves:

* Code reuse
* Hierarchical design
* Runtime polymorphism

---

## 📌 2. Syntax

```java
class Parent {
    // properties & methods
}

class Child extends Parent {
    // inherited + own features
}
```

---

## 📌 3. Basic Example

```java id="h9z7z7"
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks");
    }

    public static void main(String[] args) {
        Dog d = new Dog();
        d.sound(); // inherited
        d.bark();  // own
    }
}
```

---

## ✅ Output

```
Animal makes sound
Dog barks
```

---

## 💡 Explanation

* `Dog` inherits method `sound()` from `Animal`
* Can use both parent and its own methods

---

# 📌 4. Types of Inheritance

---

## 🔹 1. Single

```text
A → B
```

## 🔹 2. Multilevel

```text
A → B → C
```

## 🔹 3. Hierarchical

```text
   A
  / \
 B   C
```

## ❌ 4. Multiple (Not supported with classes)

* Java avoids ambiguity
* Achieved using **interfaces**

---

# 📌 5. `super` Keyword (VERY IMPORTANT)

---

## ✅ Uses

### 🔹 1. Access parent variable

```java id="wrj8rt"
super.x;
```

### 🔹 2. Call parent method

```java id="qtdap4"
super.show();
```

### 🔹 3. Call parent constructor

```java id="e2z3f3"
super();
```

---

## 📌 Example

```java id="kb9qnl"
class A {
    int x = 10;
}

class B extends A {
    int x = 20;

    void display() {
        System.out.println(super.x); // 10
        System.out.println(this.x);  // 20
    }
}
```

---

# 📌 6. Constructor in Inheritance

---

## 🔹 Rule

👉 Parent constructor runs **first**

---

### 📌 Example

```java id="24z2a2"
class A {
    A() {
        System.out.println("Parent");
    }
}

class B extends A {
    B() {
        System.out.println("Child");
    }

    public static void main(String[] args) {
        new B();
    }
}
```

---

## ✅ Output

```
Parent
Child
```

---

# 📌 7. Method Overriding (Core Part of Inheritance)

---

## 📌 Example

```java id="csq92h"
class A {
    void show() {
        System.out.println("Parent");
    }
}

class B extends A {
    void show() {
        System.out.println("Child");
    }

    public static void main(String[] args) {
        A obj = new B();
        obj.show();
    }
}
```

---

## ✅ Output

```
Child
```

---

## 💡 Explanation

* Runtime decides method → **Runtime Polymorphism**

---

# 📌 8. Access Modifiers in Inheritance

| Modifier  | Accessible in Child |
| --------- | ------------------- |
| private   | ❌ No                |
| default   | ✅ Same package      |
| protected | ✅ Yes               |
| public    | ✅ Yes               |

---

# 📌 9. Important Rules

---

* Java supports **single inheritance for classes**
* Multiple inheritance via **interfaces only**
* Constructors are **not inherited**
* Private members are **not accessible directly**

---

# 📌 10. IS-A Relationship

👉 Inheritance represents:

```text
Dog IS-A Animal
```

---

# 📌 11. Advantages

* Code reuse
* Extensibility
* Cleaner structure

---

# 📌 12. Disadvantages

* Tight coupling
* Can increase complexity

---

# 📌 13. Common Interview Traps

---

### ❗ Constructor not inherited

👉 Only called via `super()`

---

### ❗ Private methods not overridden

👉 They are not inherited

---

### ❗ Multiple inheritance confusion

👉 Not allowed with classes

---

# 🎯 Interview Summary

> Inheritance in Java is a mechanism where a child class acquires properties and behavior of a parent class using the extends keyword. It supports code reuse and runtime polymorphism, follows an IS-A relationship, and ensures that the parent constructor executes first during object creation.
