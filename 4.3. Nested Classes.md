# 🔥 Nested Classes in Java

# 📌 What are Nested Classes?

## ✅ Theory

A **nested class** is a class defined **inside another class**.

👉 Why use them?

* Logical grouping of classes
* Better **encapsulation**
* Improved readability
* Access to outer class members

---

# 📌 Types of Nested Classes

Java has **two main categories**:

---

## 🔥 1. Static Nested Class

## 🔥 2. Non-Static Nested Classes (Inner Classes)

These include:

* Member Inner Class
* Local Inner Class
* Anonymous Inner Class

---

# 🔥 1️⃣ Static Nested Class


## ✅ Theory

* Declared with `static` keyword
* Does **not depend on object of outer class**
* Can access only **static members** of outer class

---

## 📌 Code

```java
class Outer {
    static int x = 10;

    static class Inner {
        void display() {
            System.out.println(x);
        }
    }

    public static void main(String[] args) {
        Outer.Inner obj = new Outer.Inner();
        obj.display();
    }
}
```

---

## 🔑 Key Points

* No need to create outer class object
* Accessed using: `Outer.Inner`
* Memory efficient

---

# 🔥 2️⃣ Member Inner Class

## ✅ Theory

* Defined inside class but **not static**
* Requires **outer class object**
* Can access **all members** of outer class

---

## 📌 Code

```java
class Outer {
    int x = 10;

    class Inner {
        void display() {
            System.out.println(x);
        }
    }

    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner();
        inner.display();
    }
}
```

---

## 🔑 Key Points

* Has access to **private members** of outer class
* Needs outer object to be created

---

# 🔥 3️⃣ Local Inner Class

## ✅ Theory

* Declared inside a **method or block**
* Scope is limited to that method

---

## 📌 Code

```java
class Outer {
    void show() {
        class Inner {
            void display() {
                System.out.println("Local Inner Class");
            }
        }

        Inner obj = new Inner();
        obj.display();
    }

    public static void main(String[] args) {
        new Outer().show();
    }
}
```

---

## 🔑 Key Points

* Cannot be accessed outside method
* Can access **final or effectively final variables**

---

# 🔥 4️⃣ Anonymous Inner Class

## ✅ Theory

* Class **without name**
* Used for **one-time use**
* Common in:

  * Event handling
  * Interfaces
  * Abstract classes

---

## 📌 Code

```java
abstract class Animal {
    abstract void sound();
}

class Test {
    public static void main(String[] args) {
        Animal a = new Animal() {
            void sound() {
                System.out.println("Dog barks");
            }
        };

        a.sound();
    }
}
```

---

## 🔑 Key Points

* No class name
* Cannot reuse
* Compact syntax

---

# 📌 Comparison Table

| Type          | Static? | Needs Outer Object? | Scope       |
| ------------- | ------- | ------------------- | ----------- |
| Static Nested | ✅ Yes   | ❌ No                | Class level |
| Member Inner  | ❌ No    | ✅ Yes               | Class level |
| Local Inner   | ❌ No    | ✅ Yes               | Method      |
| Anonymous     | ❌ No    | ✅ Yes               | Expression  |

# 📌 Important Concepts (INTERVIEW GOLD)

---

## 🔹 Access to Outer Class Members

* Inner classes can access **private members** of outer class
  👉 Very commonly asked

---

## 🔹 Variable Rules (Local Inner Class)

```java
int x = 10;
```

👉 Must be:

* final OR effectively final

---

## 🔹 Shadowing

```java
class Outer {
    int x = 10;

    class Inner {
        int x = 20;

        void show() {
            System.out.println(this.x);      // 20
            System.out.println(Outer.this.x); // 10
        }
    }
}
```

---

## 🔹 Static Members in Inner Class

* Only **static nested class** can have static members
* Inner classes cannot have static methods (except constants)

---

## 🔹 Memory Concept

* Inner class holds reference to outer object
  👉 Extra memory usage

---

# ⚠️ Interview Edge Cases

---

### ❗ Can inner class be private?

👉 Yes

---

### ❗ Can outer class be private?

👉 No (top-level)

---

### ❗ Can we create object of inner class without outer?

👉 ❌ No (except static nested)

---

### ❗ Can interface have nested class?

👉 Yes → by default static

---

# 🎯 Real-Life Example

👉 Example: **Car Engine System**

* Outer class → Car
* Inner class → Engine

👉 Engine cannot exist independently → makes sense as inner class

# 🎯 Interview Answer (Perfect)

> Nested classes in Java are classes defined inside another class. They are used for logical grouping and better encapsulation. There are four types: static nested class, member inner class, local inner class, and anonymous inner class. Static nested classes do not require an outer object, while inner classes do and can access all members of the outer class.
