# 🔥 Tricky Constructor Concept Questions (with Answers)

## ❓ 1. Why can’t constructors be inherited?

### ✅ Answer:

Constructors are used to **initialize the current class object**, not the parent.
When a child object is created, the parent constructor is called using `super()`, so inheritance is not required.

---

## ❓ 2. Why can constructors not be overridden?

### ✅ Answer:

* Overriding requires **method inheritance**
* Constructors are **not inherited**
  👉 Hence, they cannot be overridden

---

## ❓ 3. Can constructors be overloaded?

### ✅ Answer:

Yes — constructors can be overloaded because they belong to the same class and differ in parameters.

---

## ❓ 4. Why can’t a constructor be static?

### ✅ Answer:

* Static methods belong to the **class**
* Constructors are used to create **objects (instances)**
  👉 Hence, constructor cannot be static

---

## ❓ 5. Why can’t a constructor be final?

### ✅ Answer:

* `final` prevents overriding
* Constructors are already **not overridden**
  👉 So `final` has no meaning → not allowed

---

## ❓ 6. Why can’t a constructor be abstract?

### ✅ Answer:

* Abstract methods have **no body**
* Constructors must **initialize objects**
  👉 So they must have implementation

---

## ❓ 7. Can we call constructor explicitly?

### ✅ Answer:

Yes, using:

* `this()` → same class
* `super()` → parent class

👉 Otherwise, constructors are called automatically

---

## ❓ 8. Why must `this()` be the first statement?

### ✅ Answer:

Because constructor chaining must happen **before any other initialization**, ensuring proper object setup.

---

## ❓ 9. What happens if both `this()` and `super()` are used?

### ✅ Answer:

❌ Not allowed in same constructor

👉 Reason:

* Both must be **first statement**
* Only one can exist

---

## ❓ 10. Why does parent constructor execute first?

### ✅ Answer:

Because child class depends on parent properties → parent must be initialized first.

---

## ❓ 11. What happens if parent has no default constructor?

### ✅ Answer:

Child must explicitly call:

```java
super(parameters);
```

Otherwise → **compile-time error**

---

## ❓ 12. Why is default constructor not created if parameterized constructor exists?

### ✅ Answer:

Compiler assumes developer is handling initialization manually → so it doesn’t add default constructor.

---

## ❓ 13. Can constructor have return statement?

### ✅ Answer:

* Yes → only `return;`
* ❌ Cannot return value

---

## ❓ 14. Why can’t constructor return a value?

### ✅ Answer:

Constructor’s job is **initialization**, not computation → object is returned implicitly.

---

## ❓ 15. Can we make constructor private? Why?

### ✅ Answer:

Yes — to:

* Restrict object creation
* Implement **Singleton pattern**

---

## ❓ 16. Can abstract class have constructor?

### ✅ Answer:

Yes — used to initialize common properties of subclasses.

---

## ❓ 17. Why can’t interface have constructor?

### ✅ Answer:

Interfaces don’t have object creation → so no need for constructor.

---

## ❓ 18. What is constructor chaining and why is it useful?

### ✅ Answer:

Calling one constructor from another.

👉 Benefits:

* Code reuse
* Avoid duplication
* Centralized initialization

---

## ❓ 19. What happens if constructor calls itself?

### ✅ Answer:

Leads to **infinite recursion → StackOverflowError**

---

## ❓ 20. What is the difference between constructor and initializer block?

### ✅ Answer:

| Constructor                 | Initializer Block       |
| --------------------------- | ----------------------- |
| Runs during object creation | Runs before constructor |
| Defined with class name     | No name                 |

---

## ❓ 21. Which runs first: static block, initializer block, constructor?

### ✅ Answer:

1. Static block
2. Instance initializer block
3. Constructor

---

## ❓ 22. Why is constructor not part of method inheritance?

### ✅ Answer:

Because constructor belongs to **object creation phase**, not behavior inheritance.

---

## ❓ 23. Can constructor throw exception?

### ✅ Answer:

Yes — object creation fails and exception must be handled.

---

## ❓ 24. What is the role of `this` in constructor?

### ✅ Answer:

* Refers to current object
* Resolves variable shadowing
* Calls another constructor

---

## ❓ 25. What is the role of `super` in constructor?

### ✅ Answer:

* Refers to parent class
* Calls parent constructor
* Access parent members

---

## ❓ 26. Can we have multiple constructors with same parameters?

### ❌ Answer:

No → compile-time error (same signature)

---

## ❓ 27. Why do we need constructors?

### ✅ Answer:

* Initialize objects
* Ensure object is in valid state
* Reduce manual setup

---

## ❓ 28. What is implicit constructor call?

### ✅ Answer:

If no constructor is written, compiler inserts default constructor.

---

## ❓ 29. What happens if constructor is not defined and we assign values later?

### ✅ Answer:

Object will be created with default values → then updated manually.

---

## ❓ 30. What is the difference between object initialization and declaration?

### ✅ Answer:

```java
Test t;        // declaration
t = new Test(); // initialization (constructor runs here)
```

---
