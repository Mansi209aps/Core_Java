# 🔥 Interfaces in Java

## 📌 1. Definition

An **interface** is a **blueprint of a class** that defines a **contract** (set of methods) which implementing classes must follow.

👉 Focus:

* **What to do**, not how

---

# 📌 2. Key Features

* Methods are **abstract by default** (before Java 8)
* Variables are **public static final** (constants)
* Supports **multiple inheritance**
* Cannot be instantiated

---

# 📌 3. Basic Syntax

```java id="yvx4n1"
interface InterfaceName {
    void method1();
    void method2();
}
```

---

# 📌 4. Implementation Example

```java id="rhkp69"
interface Vehicle {
    void start();
}

class Car implements Vehicle {
    public void start() {
        System.out.println("Car starts");
    }

    public static void main(String[] args) {
        Vehicle v = new Car();
        v.start();
    }
}
```

---

## 💡 Explanation

* `Vehicle` defines method
* `Car` must implement it
* Object created using **interface reference (upcasting)**

---

# 📌 5. Types of Methods in Interface (Java 8+)

## 🔹 1. Abstract Method

```java id="cg89yf"
void show();
```

---

## 🔹 2. Default Method

```java id="7d0q40"
default void display() {
    System.out.println("Default method");
}
```

👉 Can have body

---

## 🔹 3. Static Method

```java id="4vtvta"
static void print() {
    System.out.println("Static method");
}
```

👉 Called using interface name

---

## 🔹 4. Private Method (Java 9+)

```java id="59scqy"
private void helper() {}
```

👉 Used internally inside interface

---

# 📌 6. Multiple Inheritance using Interface

```java id="btdzdm"
interface A {
    void show();
}

interface B {
    void display();
}

class Test implements A, B {
    public void show() {}
    public void display() {}
}
```

👉 One class implements multiple interfaces

---

# 📌 7. Interface vs Abstract Class

| Feature     | Interface           | Abstract Class      |
| ----------- | ------------------- | ------------------- |
| Methods     | Mostly abstract     | Abstract + concrete |
| Variables   | public static final | Any type            |
| Inheritance | Multiple            | Single              |

---

# 📌 8. Important Rules

* Class must implement **all methods**
* Otherwise → class must be `abstract`
* Methods must be **public** when implemented
* Cannot create object of interface

---

# 📌 9. Functional Interface (Important)

## ✅ Definition

Interface with **only one abstract method**

---

### 📌 Example

```java id="bg2t6f"
@FunctionalInterface
interface Test {
    void run();
}
```

👉 Used in **lambda expressions**

---

# 📌 10. Marker Interface

## ✅ Definition

Interface with **no methods**

👉 Example:

* `Serializable`
* `Cloneable`

---

# 📌 11. Real-Life Example

👉 Example: Remote Control

* Interface → buttons (methods)
* TV → implementation

---

# 📌 12. Advantages

* Achieves abstraction
* Supports multiple inheritance
* Loose coupling
* Flexibility

---

# 📌 13. Common Interview Traps

### ❗ Interface variables are always constants

### ❗ Cannot use constructor

### ❗ Methods are public by default

### ❗ Static methods are NOT inherited

---

# 🎯 Interview Summary

> An interface in Java is a blueprint that defines a contract for classes. It supports abstraction and multiple inheritance. Interfaces can contain abstract, default, static, and private methods, and are widely used for designing flexible and loosely coupled systems.
