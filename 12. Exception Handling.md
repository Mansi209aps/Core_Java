# 🔷 1. What is Exception Handling?

* Mechanism to handle **runtime errors** so program doesn’t crash
* Maintains **normal flow of execution**

👉 Example:

```java
int a = 10 / 0; // ArithmeticException
```

---

# 🔷 2. Exception Hierarchy (VERY IMPORTANT 🔥)

```
Throwable
 ├── Error
 └── Exception
      ├── Checked Exception
      └── Unchecked Exception (RuntimeException)
```

---

## 🔸 Error

* Serious issues (JVM related)
* Cannot be handled usually
  👉 Example: `OutOfMemoryError`

---

## 🔸 Exception

Handled by programmer

### ✔️ Checked Exceptions (Compile-time)

* Must be handled or declared
  👉 Examples:
* `IOException`
* `SQLException`

---

### ✔️ Unchecked Exceptions (Runtime)

* Occur at runtime
  👉 Examples:
* `NullPointerException`
* `ArithmeticException`
* `ArrayIndexOutOfBoundsException`

---

# 🔷 3. Keywords in Exception Handling

| Keyword   | Purpose                       |
| --------- | ----------------------------- |
| `try`     | Code that may cause exception |
| `catch`   | Handles exception             |
| `finally` | Always executes               |
| `throw`   | Explicitly throw exception    |
| `throws`  | Declare exception             |

---

# 🔷 4. try-catch Block

```java
try {
    int a = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero");
}
```

👉 Flow:

* Exception occurs → JVM creates object → transferred to `catch`

---

# 🔷 5. Multiple catch Blocks

```java
try {
    int arr[] = new int[5];
    arr[10] = 50;
} catch (ArithmeticException e) {
} catch (ArrayIndexOutOfBoundsException e) {
}
```

👉 Rule:

* **Specific → General** (child first)

---

# 🔷 6. finally Block

```java
try {
    int a = 10 / 2;
} finally {
    System.out.println("Always executed");
}
```

👉 Executes:

* Whether exception occurs or not
* Even after `return` (except JVM crash)

---

# 🔷 7. throw Keyword

```java
throw new ArithmeticException("Error");
```

👉 Used to **manually throw exception**

---

# 🔷 8. throws Keyword

```java
void readFile() throws IOException {
}
```

👉 Declares exception to caller

---

# 🔷 9. Checked vs Unchecked (INTERVIEW FAVORITE ⭐)

| Feature    | Checked      | Unchecked        |
| ---------- | ------------ | ---------------- |
| Checked at | Compile time | Runtime          |
| Handling   | Mandatory    | Optional         |
| Class      | Exception    | RuntimeException |

---

# 🔷 10. Custom Exception

```java
class MyException extends Exception {
    MyException(String msg) {
        super(msg);
    }
}
```

👉 Used for **business logic errors**

---

# 🔷 11. Exception Propagation

```java
void m1() {
    m2();
}
void m2() {
    int a = 10 / 0;
}
```

👉 Exception moves **up the call stack**

---

# 🔷 12. try-with-resources (Java 7+)

```java
try (FileReader fr = new FileReader("file.txt")) {
}
```

👉 Automatically closes resources
👉 Must implement **AutoCloseable**

---

# 🔷 13. Important Methods of Exception Class

| Method              | Purpose        |
| ------------------- | -------------- |
| `printStackTrace()` | Detailed error |
| `getMessage()`      | Error message  |
| `toString()`        | Exception info |

---

# 🔷 14. Final vs Finally vs Finalize (VERY COMMON ❗)

| Term         | Meaning                         |
| ------------ | ------------------------------- |
| `final`      | Keyword (constant, no override) |
| `finally`    | Block (always executes)         |
| `finalize()` | Method (called by GC)           |

# 🔷 15. Common Interview Scenarios

## ✔️ Case 1: No catch

```java
try {
    int a = 10 / 0;
}
```

👉 Runtime crash

---

## ✔️ Case 2: Multiple catch order

❌ Wrong:

```java
catch (Exception e)
catch (ArithmeticException e)
```

✔️ Correct:

```java
catch (ArithmeticException e)
catch (Exception e)
```

---

## ✔️ Case 3: finally with return

```java
int test() {
    try {
        return 1;
    } finally {
        return 2;
    }
}
```

👉 Output: **2**

---

# 🔷 16. Best Practices (INTERVIEW READY ✅)

* Catch **specific exceptions**
* Avoid empty catch blocks
* Use `finally` or try-with-resources
* Don’t use exceptions for normal flow
* Log exceptions properly

---

# 🔷 17. One-Line Interview Answer 🎯

👉 *"Exception handling in Java is a mechanism to handle runtime errors using try-catch, ensuring smooth program execution without termination."*
