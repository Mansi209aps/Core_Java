# 🔥 Pass by Value in Java

## 📌 1. Definition

Java is **strictly pass-by-value**.

👉 This means:

* A **copy of the variable (or reference)** is passed to methods
* Original variable is **not directly modified**

---

# 📌 2. Key Idea (Most Important)

👉 Java never passes:

* actual variable
* actual object

👉 It always passes:

* **copy of value**
* For objects → **copy of reference**

---

# 📌 3. Primitive Type Example

```java
class Test {

    void change(int x) {
        x = 100;
    }

    public static void main(String[] args) {
        int a = 10;
        new Test().change(a);
        System.out.println(a);
    }
}
```

## ✅ Output

```
10
```

## 💡 Explanation

* Copy of `a` is passed → `x`
* Changing `x` does NOT affect original `a`

---

# 📌 4. Object Example (Important)

```java
class Test {
    int x = 10;

    void change(Test t) {
        t.x = 100;
    }

    public static void main(String[] args) {
        Test obj = new Test();
        obj.change(obj);
        System.out.println(obj.x);
    }
}
```

## ✅ Output

```
100
```

## 💡 Explanation

* Copy of reference is passed
* Both refer to **same object in heap**
* So changes reflect

---

# 📌 5. Tricky Case (Interview Favorite)

```java
class Test {
    int x = 10;

    void change(Test t) {
        t = new Test(); // new object
        t.x = 200;
    }

    public static void main(String[] args) {
        Test obj = new Test();
        obj.change(obj);
        System.out.println(obj.x);
    }
}
```

## ✅ Output

```
10
```

## 💡 Explanation

* New object created inside method
* Only local copy of reference changes
* Original object remains unchanged

---

# 📌 6. Visual Understanding

👉 For object:

```
Before:
obj → Object1 (x=10)

Inside method:
t → Object1 (copy of reference)

Modify:
t.x = 100 → affects Object1
```

---

# 📌 7. Important Points

* Java is **NOT pass-by-reference**
* Primitive → copy of value
* Object → copy of reference
* Object state can change, reference cannot

---

# 📌 8. Common Confusion

❌ “Java passes objects by reference”
✔ Correct: Java passes **reference by value**

---

# 🎯 Interview Summary

> Java uses pass-by-value where a copy of the variable is passed to methods. For primitives, the value is copied, so changes don’t affect the original variable. For objects, a copy of the reference is passed, allowing modification of the object’s state but not the original reference itself.
