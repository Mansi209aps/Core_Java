# 🔥 Method Overriding


## 📌 1. Definition

Method Overriding occurs when a **subclass provides its own implementation of a method that is already defined in the parent class**.

👉 It is used to achieve **runtime polymorphism**.

---

## 📌 2. Basic Example

```java id="j0n0b2"
class A {
    void show() {
        System.out.println("Parent method");
    }
}

class B extends A {
    @Override
    void show() {
        System.out.println("Child method");
    }

    public static void main(String[] args) {
        A obj = new B();
        obj.show();
    }
}
```

---

## ✅ Output

```
Child method
```

---

## 💡 Explanation

* Parent reference (`A obj`) points to child object (`new B()`)
* JVM decides method at **runtime**
* Child method overrides parent method

---

# 📌 3. Rules of Method Overriding

---

## ✅ Must Follow

* Same method name
* Same parameters
* Same or **covariant return type**
* Cannot reduce access modifier

---

## ❌ Not Allowed

* Cannot override:

  * `final` methods
  * `static` methods (they are **hidden**, not overridden)
  * `private` methods

---

# 📌 4. Access Modifier Rule

```java
class A {
    public void show() {}
}

class B extends A {
    protected void show() {} // ❌ ERROR (reducing visibility)
}
```

---

# 📌 5. Overriding vs Overloading

| Feature      | Overriding   | Overloading  |
| ------------ | ------------ | ------------ |
| Class        | Parent-child | Same class   |
| Polymorphism | Runtime      | Compile-time |
| Parameters   | Same         | Different    |

---

# 🎯 Interview Summary

> Method overriding in Java allows a subclass to provide a specific implementation of a method already defined in its parent class. It follows rules like same method signature and supports runtime polymorphism, where method execution is determined at runtime based on the object type.
