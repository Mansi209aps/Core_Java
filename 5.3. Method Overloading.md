# 🔥 Method Overloading (Compile-Time Polymorphism)


## 📌 1. Definition

Method Overloading means **defining multiple methods with the same name but different parameter lists** within the same class.

👉 It improves:

* Readability
* Flexibility
* Code reuse

---

## 📌 2. Basic Example

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }

    double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        Calculator c = new Calculator();

        System.out.println(c.add(2, 3));       // 5
        System.out.println(c.add(2, 3, 4));    // 9
        System.out.println(c.add(2.5, 3.5));   // 6.0
    }
}
```

### 💡 Explanation

* Same method name `add`
* Different parameters → Java chooses method at **compile time**

---

# 📌 3. Rules of Overloading

---

## ✅ Allowed (Valid Overloading)

### 🔹 Change in number of parameters

```java
void show(int a)
void show(int a, int b)
```

### 🔹 Change in type of parameters

```java
void show(int a)
void show(double a)
```

### 🔹 Change in order of parameters

```java
void show(int a, double b)
void show(double a, int b)
```

---

## ❌ Not Allowed

### 🔹 Only changing return type

```java
int show()
double show() // ❌ ERROR
```

👉 Reason: Method signature = name + parameters only

---

# 📌 4. Method Signature

👉 Signature includes:

* Method name
* Parameter list

👉 Does NOT include:

* Return type
* Access modifier

---

# 📌 5. How Java Resolves Overloading (VERY IMPORTANT)

---

### Priority Order:

1. **Exact Match**
2. **Widening** (int → long → float → double)
3. **Autoboxing** (int → Integer)
4. **Varargs**

---

## 📌 Example

```java
class Test {

    void show(int a) {
        System.out.println("int");
    }

    void show(double a) {
        System.out.println("double");
    }

    public static void main(String[] args) {
        new Test().show(10);
    }
}
```

### ✅ Output

```
int
```

👉 Exact match preferred over widening

---

# 📌 6. Widening Example

```java
class Test {
    void show(double a) {
        System.out.println("double");
    }

    public static void main(String[] args) {
        new Test().show(10); // int → double
    }
}
```

### ✅ Output

```
double
```

---

# 📌 7. Autoboxing Example

```java
class Test {
    void show(Integer a) {
        System.out.println("Integer");
    }

    public static void main(String[] args) {
        new Test().show(10);
    }
}
```

### ✅ Output

```
Integer
```

---

# 📌 8. Varargs Example

```java
class Test {
    void show(int... a) {
        System.out.println("Varargs");
    }

    public static void main(String[] args) {
        new Test().show(1, 2, 3);
    }
}
```

---

# 📌 9. Priority Example (Very Tricky)

```java
class Test {

    void show(int a) {
        System.out.println("int");
    }

    void show(Integer a) {
        System.out.println("Integer");
    }

    void show(int... a) {
        System.out.println("Varargs");
    }

    public static void main(String[] args) {
        new Test().show(10);
    }
}
```

### ✅ Output

```
int
```

👉 Priority: Exact > Autoboxing > Varargs

---

# 📌 10. Overloading with Constructor

```java
class Test {
    Test() {}

    Test(int a) {}
}
```

👉 Constructors can also be overloaded

---

# 📌 11. Key Differences: Overloading vs Overriding

| Feature      | Overloading  | Overriding   |
| ------------ | ------------ | ------------ |
| Occurs in    | Same class   | Parent-child |
| Polymorphism | Compile-time | Runtime      |
| Parameters   | Must differ  | Must be same |

---

# 📌 12. Advantages

* Improves readability
* Avoids multiple method names
* Enables flexible APIs

---

# ⚠️ Common Mistakes

---

### ❗ Confusing return type

👉 Return type alone ≠ overloading

---

### ❗ Ambiguous calls

```java
void show(int a, double b)
void show(double a, int b)

show(10, 10); // ❌ ambiguous
```

---

# 🎯 Interview Summary (Perfect Answer)

> Method overloading in Java is a feature where multiple methods have the same name but different parameter lists. It enables compile-time polymorphism, allowing methods to behave differently based on input. Java resolves overloaded methods using priority: exact match, widening, autoboxing, and varargs.
