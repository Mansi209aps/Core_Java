# 🔷 1. What is Multithreading?

* Executing **multiple threads simultaneously** within a program
* Thread = lightweight sub-process

👉 Benefits:

* Better performance
* Efficient CPU utilization


# 🔷 2. Process vs Thread (BASIC INTERVIEW 🔥)

| Feature       | Process  | Thread               |
| ------------- | -------- | -------------------- |
| Memory        | Separate | Shared               |
| Overhead      | High     | Low                  |
| Communication | IPC      | Easy (shared memory) |


# 🔷 3. Ways to Create Thread ⭐

## ✔️ 1. Extending Thread class

```java id="1j89ze"
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running");
    }
}
```


## ✔️ 2. Implementing Runnable (Recommended)

```java id="4k1tfm"
class MyTask implements Runnable {
    public void run() {
        System.out.println("Task running");
    }
}
```


## ✔️ 3. Using Lambda (Java 8+)

```java id="6w4ylu"
new Thread(() -> System.out.println("Thread")).start();
```


# 🔷 4. Thread Lifecycle ⭐

```text id="rzmb9y"
New → Runnable → Running → Blocked/Waiting → Terminated
```


# 🔷 5. Important Thread Methods

| Method        | Purpose                 |
| ------------- | ----------------------- |
| `start()`     | Starts thread           |
| `run()`       | Thread logic            |
| `sleep()`     | Pause thread            |
| `join()`      | Wait for thread         |
| `yield()`     | Pause for other threads |
| `interrupt()` | Interrupt thread        |


# 🔷 6. Synchronization (CRITICAL 🔥)

## 🔹 Problem: Race Condition

* Multiple threads access shared data → inconsistent result


## 🔹 Solution: Synchronization

### ✔️ Synchronized Method

```java id="a1r0uv"
synchronized void increment() {
    count++;
}
```


### ✔️ Synchronized Block

```java id="2r2kj9"
synchronized(this) {
    count++;
}
```


## 🔹 Key Points

* Only one thread executes at a time
* Uses **intrinsic lock (monitor)**


# 🔷 7. Inter-Thread Communication ⭐

## ✔️ Methods:

* `wait()`
* `notify()`
* `notifyAll()`

```java id="s9b1hp"
synchronized(obj) {
    obj.wait();
    obj.notify();
}
```

👉 Used for coordination between threads


# 🔷 8. Deadlock (IMPORTANT ❗)

## 🔹 Definition

* Two threads waiting for each other forever


## 🔹 Example

```java id="z4u1y2"
Thread 1 → lock A → waiting for B
Thread 2 → lock B → waiting for A
```


## 🔹 Prevention

* Avoid nested locks
* Use lock ordering
* Use `tryLock()`


# 🔷 9. Thread Safety

* Code that works correctly in multithreaded environment

👉 Achieved by:

* Synchronization
* Immutable objects
* Atomic classes


# 🔷 10. Volatile Keyword ⭐

```java id="a9hffg"
volatile int flag;
```

👉 Ensures:

* **Visibility** (latest value across threads)
* Not for atomicity


# 🔷 11. Executor Framework (VERY IMPORTANT 🔥)

## 🔹 Why?

* Better thread management
* Avoid manual thread handling


## 🔹 Example

```java id="5vud3r"
ExecutorService ex = Executors.newFixedThreadPool(2);
ex.submit(() -> System.out.println("Task"));
ex.shutdown();
```


## 🔹 Types

* Fixed Thread Pool
* Cached Thread Pool
* Single Thread Executor


# 🔷 12. Callable & Future ⭐

## ✔️ Callable

* Returns value

```java id="l35v1u"
Callable<Integer> task = () -> 10;
```


## ✔️ Future

* Holds result

```java id="k3xk1n"
Future<Integer> result = ex.submit(task);
result.get();
```


# 🔷 13. Locks (Advanced)

## ✔️ ReentrantLock

```java id="k6xpf2"
Lock lock = new ReentrantLock();
lock.lock();
lock.unlock();
```

👉 More control than `synchronized`


# 🔷 14. Concurrent Collections ⭐

* Thread-safe collections

Examples:

* `ConcurrentHashMap`
* `CopyOnWriteArrayList`


# 🔷 15. Atomic Classes

* Used for thread-safe operations without synchronization

```java id="5pdr34"
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();
```


# 🔷 16. Fork/Join Framework

* Used for parallel processing
* Break task → smaller subtasks


# 🔷 17. Thread Pool vs Manual Threads

| Feature       | Thread Pool | Manual |
| ------------- | ----------- | ------ |
| Performance   | High        | Low    |
| Resource mgmt | Efficient   | Poor   |


# 🔷 18. Common Problems in Multithreading

* Race condition
* Deadlock
* Starvation
* Livelock


# 🔷 19. Best Practices ✅

* Prefer **ExecutorService**
* Avoid excessive synchronization
* Use **immutable objects**
* Use concurrent collections
* Always release locks


# 🔷 20. Interview Tricky Points 🔥

* `start()` vs `run()`
* `sleep()` vs `wait()`
* `notify()` vs `notifyAll()`
* `volatile` vs `synchronized`


# 🔷 21. One-Line Interview Summary 🎯
