# 🔥 Access Modifiers in Methods

Access modifiers define **who can access a method**.

---

## 📌 Types of Access Modifiers

| Modifier               | Access Level              |
| ---------------------- | ------------------------- |
| `private`              | Only within same class    |
| `default` (no keyword) | Within same package       |
| `protected`            | Same package + subclasses |
| `public`               | Everywhere                |

---

## 📌 Example

```java
class Test {

    private void privateMethod() {
        System.out.println("Private");
    }

    void defaultMethod() {
        System.out.println("Default");
    }

    protected void protectedMethod() {
        System.out.println("Protected");
    }

    public void publicMethod() {
        System.out.println("Public");
    }
}
```

---

## 💡 Explanation

* `privateMethod()` → accessible only inside `Test`
* `defaultMethod()` → accessible within package
* `protectedMethod()` → accessible in subclasses (even outside package)
* `publicMethod()` → accessible everywhere

# 🔥 Method Signature

## ✅ Definition

Method signature = **method name + parameter list**

👉 Return type is NOT part of signature

---

## 📌 Example

```java
void show(int a)
void show(int a, int b)  // valid (overloading)
```

# 🔥 Method Modifiers (Complete List)

---

## 📌 1. static

* Belongs to class
* Called without object

---

## 📌 2. final

* Cannot be overridden

---

## 📌 3. abstract

* No body
* Must be implemented

## 📌 4. synchronized

### ✅ Theory

# 🔥 Method Overriding (IMPORTANT ADDITION)

---

## ✅ Definition

Subclass provides **specific implementation** of parent method

---

## 📌 Rules

* Same method name
* Same parameters
* Return type same or covariant
* Cannot reduce access level

---

## 📌 Example

```java
class A {
    void show() {
        System.out.println("Parent");
    }
}

class B extends A {
    @Override
    void show() {
        System.out.println("Child");
    }

    public static void main(String[] args) {
        A obj = new B();
        obj.show();
    }
}
```

---

## ✅ Output

```
Child
```

---

## 💡 Explanation

* Runtime decides method → **Runtime Polymorphism**

---

# 🔥 Covariant Return Type

---

## ✅ Theory

Child method can return **subtype** of parent return type

---

## 📌 Example

```java
class A {}

class B extends A {}

class Test {
    A get() {
        return new A();
    }
}

class Demo extends Test {
    B get() {
        return new B();
    }
}
```

---

# 🔥 Varargs (Variable Arguments)

---

## ✅ Theory

Allows method to accept **multiple arguments**

---

## 📌 Code

```java
class Test {
    void show(int... nums) {
        for (int n : nums) {
            System.out.println(n);
        }
    }
}
```

---

## 💡 Explanation

* Internally treated as array

---

# 🔥 Method Hiding (Static Methods)

---

## ✅ Theory

* Static methods are **not overridden**, they are **hidden**

---

## 📌 Example

```java
class A {
    static void show() {
        System.out.println("A");
    }
}

class B extends A {
    static void show() {
        System.out.println("B");
    }
}
```

---

## 💡 Explanation

* Depends on reference type, not object

---

# 🔥 Important Rules for Methods

---

### ✔ Allowed

* Overloading
* Overriding
* Recursion
* Returning objects

---

### ❌ Not Allowed

* Duplicate method signature
* Abstract method in non-abstract class
* Static method overriding (only hiding)

---

# 🔥 Access Modifier + Overriding Rule (IMPORTANT)

---

## 📌 Rule:

Child class **cannot reduce visibility**

---

### ❌ Example

```java
class A {
    public void show() {}
}

class B extends A {
    private void show() {} // ❌ ERROR
}
```

# 🎯 Final Interview Answer (Complete)

> Methods in Java are reusable blocks of code that perform specific tasks. They support features like access control using modifiers (private, public, protected, default), overloading, overriding, recursion, and varargs. Methods can be static or instance-based, and Java strictly follows pass-by-value. Proper understanding of method signatures, access modifiers, and polymorphism is essential for writing efficient and maintainable programs.
